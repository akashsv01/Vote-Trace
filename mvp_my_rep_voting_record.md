# MVP Spec: My Rep's Voting Record, Explained

## One-liner

Enter your address, see your actual representatives and their recent votes explained in plain language, nonpartisan.

---

## Stack

- React frontend (single page)
- Python (FastAPI) backend
- Anthropic API (Claude) for vote explanations
- Google Civic Information API for rep lookup by address
- ProPublica Congress API for federal voting records

---

## User Flow

1. User lands on page, sees a single address input field
2. They enter their address and submit
3. App returns their federal representatives (House + Senate) with name, title, party, and photo
4. User clicks a rep to see their 10 most recent votes
5. Each vote shows: bill name, date, how they voted (Yes/No/Not Voting), and a 2-3 sentence plain-language explanation of what the bill would do and why it matters
6. Explanations are generated by Claude on demand (click to expand), not pre-loaded

---

## API Keys Needed

- Google Civic Information API key (free, Google Cloud Console)
- ProPublica Congress API key (free, email request, instant)
- Anthropic API key

---

## Backend Endpoints

### POST /reps

- **Input:** `{ address: string }`
- Calls Google Civic API with the address
- **Returns:** list of federal reps with name, party, state, district, bioguide_id, photo_url

### GET /votes/{bioguide_id}

- Calls ProPublica Congress API for recent votes by that member
- **Returns:** last 10 votes with bill_id, bill_title, date, vote_position, bill_description

### POST /explain

- **Input:** `{ bill_title: string, bill_description: string, vote_position: string, rep_name: string }`
- Calls Claude API
- **System prompt:** "You are a nonpartisan civic education tool. Explain legislation in plain language. Never editorialize or imply a vote was good or bad. Be factual and neutral."
- **Returns:** 2-3 sentence explanation

---

## Frontend Components

| Component | Responsibility |
|---|---|
| `AddressInput` | Form with single input and submit button |
| `RepCard` | Photo, name, party, title, click to expand |
| `VoteList` | List of 10 votes per rep |
| `VoteItem` | Bill name, date, vote badge (Yes/No/Abstain), expandable explanation |
| `ExplainButton` | Triggers POST /explain and renders result inline |

---

## What to Cut if Time is Short

- State and local reps (Google Civic returns them but ProPublica only covers federal -- skip non-federal for the demo)
- Caching (call APIs fresh every time for the hackathon)
- Auth, accounts, saved reps -- none of it
- Mobile optimization -- get it working on desktop first

---

## Demo Script (for judging)

1. Enter a real address (use the hackathon venue's address)
2. Show the two senators and house rep that appear
3. Click one senator, show their recent votes
4. Expand a vote, show Claude's plain-language explanation generate live
5. Emphasize: "Just ask Claude" cannot tell you how your specific senator voted on this specific bill last week

---

## The One Thing That Has to Work

The address-to-rep lookup plus at least one vote with a live Claude explanation. If the ProPublica integration breaks, hardcode one rep's votes as a fallback so the Claude explanation layer still demos cleanly.

---

## Why This Passes the "Just Ask Claude" Test

| Leg | How this app satisfies it |
|---|---|
| Structured data integration | Google Civic API (address-to-rep) + ProPublica (live voting records) -- Claude alone has neither |
| Workflow | Address input → rep lookup → vote list → on-demand explanation; a guided process, not a blank chat |
| Structured output | Vote badges, rep cards, expandable explanations -- a real UI, not a wall of text |
| Specific audience | Disengaged voters who would never open claude.ai but might use a purpose-built civic tool |
